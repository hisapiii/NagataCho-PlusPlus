# NagataCho++ (NagataCho Plus Plus)

NagataCho++は、日本の政治的慣習（建前、先送り、責任転嫁、忖度）を仕様として組み込んだ、動的型付けインタプリタ言語です。
Python上で動作し、極めて可読性が高い一方で、実行結果が政治情勢により予測不能になる特徴を持ちます。

## 概要

この言語は「論理的な正しさ」よりも「その場の空気」を重視します。
ループ処理は「問題の先送り」として定義され、条件分岐は「善処」として実装されています。

## 動作環境

* Python 3.x
* 実行エンジン: main.py

## 実行方法

ソースコード（拡張子 .ncp 推奨）を用意し、以下のコマンドで審議（実行）します。

$ python main.py

## 言語仕様

### 1. 基本データ型と定数

* Yosan (予算): 数値型。変数宣言に使用します。
* Touben (答弁): 文字列型。
* KakugiKettei (閣議決定): 真 (True)。この言語において絶対的な真実です。
* Gojin (誤認): 偽 (False)。
* Kentou (検討): 何もしない (Null/Void)。「前向きに検討します」とログを出力しますが、処理は行われません。

### 2. 入出力

* KishaKaiken("メッセージ"): 標準出力。記者会見を行います。
* PublicComment(): 標準入力。国民からの意見を受け付けますが、約20%の確率で「誤認」として破棄（シュレッダー）されます。

### 3. 制御構文

* Sakiokuri (条件) { ... }:
  Whileループ。問題が解決（条件が偽になる）するまで、処理を永遠に先送りします。

* Zensho (条件) -> 実行文:
  簡易If文。条件に合致する場合のみ、前向きに善処（実行）します。

* Habatsu [新派閥] extends/Riritou [旧党名]:
  派閥結成や離党を宣言します（ログ出力のみ）。

### 4. 特別コマンド

* Jinin [対象]:
  特定のモード（Koizumiなど）を強制終了させます。

## 標準ライブラリ・特殊モード

NagataCho++には、特定の政治家の挙動を模倣するモードが搭載されています。

### Koizumi モジュール
import Koizumi で有効化。
すべての「KishaKaiken（出力）」がトートロジー（同義反復）形式、またはポエム調に変換されます。
解除するには Jinin Koizumi を実行してください。

例: 「今のままではいけないと思います。だからこそ、今のままではいけないと思います。」

### Renho モジュール (#pragma Renho)
事業仕分けモード。
数値計算において「1（1位）」が発生した場合、自動的に「2（2位）」に書き換えられます。

### Abe モジュール (SoExcited)
スロットリング機能。
処理がヒートアップした際、SoExcited コマンドを実行することで、処理を3秒間停止させ、野次を鎮めます。

### Ryutaro モジュール (RyutaroCatch)
致命的エラーハンドラ。
プログラムの継続が困難になった際、号泣しながら強制終了します。

## サンプルコード

// 小泉構文のテスト
import Koizumi
Sexy Touben msg = "プラスチックの原料は石油です"
KishaKaiken(msg)
Jinin Koizumi

// 先送りループ
Yosan tax = 8
Sakiokuri (tax < 11) {
    KishaKaiken("検討中... 税率: " + tax + "%")
    tax++
}

// 強制終了
RyutaroCatch
